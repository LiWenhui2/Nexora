# NaiwaProxy 节点管理使用说明

本文档说明桌面端 **节点管理** 页面目前已经实现的功能、使用方法，以及各项配置的含义。

## 入口与界面

左侧导航包含两个主要入口：

| 入口 | 作用 |
|------|------|
| 节点管理 | 选择节点、测速、启用代理、查看流量、配置系统代理/TUN/路由 |
| 订阅管理 | 打开节点导入窗口，支持粘贴分享链接或订阅地址 |

右上角 **启用代理** 是主开关。打开后会启动 Xray Core，并按当前系统代理、TUN、路由规则配置生效；关闭后会停止 Core、停止 TUN，并清理自动配置的系统代理。

## 节点列表

节点表格展示以下字段：

| 字段 | 含义 |
|------|------|
| 名称 | 节点名称；当前活动节点前会显示星标 |
| 协议 | 当前支持 VMess |
| 地址 | 节点服务器地址和端口 |
| 地区 | 预留字段，当前未自动识别 |
| TCP | TCP 连接延迟 |
| 订阅 | 手动导入或订阅来源域名 |
| 更新 | 订阅导入时间，如“刚刚”“2 小时前” |

右键节点可执行：

| 操作 | 说明 |
|------|------|
| 设为活动节点 | 将该节点设为当前使用节点；如果 Core 正在运行，会自动重启 Core 生效 |
| 编辑节点 | 打开节点编辑弹窗 |
| TCP 测速 | 只测试该节点 TCP 连通性 |
| 删除节点 | 删除选中节点 |

列表空白处右键可执行：

| 操作 | 说明 |
|------|------|
| 新建节点 | 手动添加 VMess 节点 |
| 导入节点 | 批量导入 vmess 链接或订阅 |
| 全部 TCP 测速 | 对所有节点执行 TCP 测速 |
| 节点维护 | 预留功能，当前仅提示 |

## TCP 测速与切换最快

**TCP 测速** 只检查服务器地址和端口是否能连上，并显示连接耗时。

| 显示 | 含义 |
|------|------|
| `-` | 未测速 |
| `...` | 测速中 |
| `Timeout` | 超时或无法连接 |
| `xx ms` | TCP 连接耗时 |

**切换最快** 会从已有 TCP 测速结果中选择延迟最低的节点，并设为当前节点。如果 Core 正在运行，会自动重启 Core 使新节点生效。

## 节点导入与订阅管理

左侧 **订阅管理** 和节点列表右键 **导入节点** 都会打开同一个导入窗口。

支持输入：

| 输入类型 | 说明 |
|----------|------|
| `vmess://` 单条链接 | 一行一条 |
| 多条 `vmess://` 链接 | 批量导入 |
| Base64 订阅正文 | 自动解码并提取 vmess |
| `http://` / `https://` 订阅地址 | 自动下载订阅并解析 |

导入结果：

| 情况 | 表格显示 |
|------|----------|
| 手动粘贴 vmess 链接 | 订阅列显示“手动” |
| 通过订阅 URL 导入 | 订阅列显示订阅域名 |
| 成功导入后 | 更新列显示导入时间 |

当前仅支持 VMess。VLESS、Trojan、Shadowsocks、Clash YAML、Sing-box JSON 等格式尚未实现。

## 系统代理

系统代理控制 Windows 当前用户的 Internet 代理设置。它不单独启动 Core，须配合右上角 **启用代理** 使用。

默认端口：

| 类型 | 地址 |
|------|------|
| HTTP | `127.0.0.1:10809` |
| SOCKS | `127.0.0.1:10808` |

选项说明：

| 选项 | 含义 |
|------|------|
| 清除系统代理 | 关闭系统代理，并清理 `ProxyServer` 和 `AutoConfigURL` |
| 自动配置系统代理 | Core 运行时设置 HTTP 代理为 `127.0.0.1:10809` |
| 不改变系统代理 | 不修改 Windows 系统代理配置 |
| PAC 模式 | 生成 `%APPDATA%\NaiwaProxy\proxy.pac`，局域网/本地地址直连，其余走 HTTP 代理 |

关闭代理或退出应用时，如果当前模式是自动配置或 PAC，会自动清除系统代理。

## TUN 模式

TUN 模式通过 `sing-box` 创建虚拟网卡，将系统流量转发到本地 SOCKS 端口。

依赖文件位于安装目录 `cores`：

| 文件 | 作用 |
|------|------|
| `wintun.dll` | Windows TUN 驱动 |
| `sing-box.exe` | TUN 运行时 |
| `libcronet.dll` | sing-box 随包依赖 |

使用条件：

| 条件 | 说明 |
|------|------|
| 需要管理员权限 | 点击 TUN 开关时，如果当前不是管理员，会请求 UAC 并以管理员身份重启 |
| 需要先启用代理 | TUN 会转发到本地 SOCKS，所以 Core 必须先运行 |
| 关闭代理会停止 TUN | 避免虚拟网卡残留 |

状态说明：

| 状态 | 含义 |
|------|------|
| 已关闭 | TUN 未启用 |
| 待启动 | 配置已勾选，但当前运行时未启动 |
| 运行中 | sing-box TUN 已启动 |
| 需要管理员 | 当前进程没有管理员权限 |
| 缺少运行时 | `sing-box.exe` 或依赖文件缺失 |

## 路由规则

路由规则决定流量走代理、直连或阻止。修改路由规则后，如果 Core 正在运行，程序会自动重启 Core 生效，界面不再提供手动“重启 Core”按钮。

内置模式：

| 模式 | 含义 |
|------|------|
| 全局代理 | 默认全部流量走代理 |
| V4-绕过大陆 (Whitelist) | 私有 IP、`geoip:cn`、`geosite:cn` 直连，其余走代理 |
| 绕过局域网 | 私有 IP、本地网段直连，其余走代理 |
| 直连模式 | 所有流量直连，相当于不代理 |
| 自定义规则 | 使用“编辑规则”弹窗中的自定义列表 |

### 自定义规则

点击 **编辑规则** 可打开自定义路由弹窗。弹窗分为六个列表，每行一条规则。

| 列表 | 出站 |
|------|------|
| 走代理域名 | proxy |
| 直连域名 | direct |
| 阻止域名 | block |
| 走代理 IP / CIDR | proxy |
| 直连 IP / CIDR | direct |
| 阻止 IP / CIDR | block |

匹配优先级：

1. 阻止
2. 直连
3. 代理

域名支持：

| 写法 | 含义 |
|------|------|
| `example.com` | 自动转换为 `domain:example.com` |
| `domain:example.com` | 匹配域名及子域 |
| `full:example.com` | 完整域名匹配 |
| `regexp:^.*\.example\.com$` | 正则匹配 |
| `geosite:category-ads-all` | Xray geosite 列表 |

IP 支持：

| 写法 | 含义 |
|------|------|
| `1.2.3.4` | 单个 IP |
| `1.2.3.0/24` | CIDR 网段 |
| `geoip:cn` | Xray geoip 列表 |

保存自定义规则后，路由模式会自动切换为 **自定义规则**。

## 上传下载与累计流量

界面展示：

| 字段 | 含义 |
|------|------|
| 顶部下行/上传 | 当前实时速度 |
| 当前下载 | 当前下载速度 |
| 当前上传 | 当前上传速度 |
| 累计流量 | 历史累计上传 + 下载 |

流量来源为 Xray Stats API。程序会定时读取代理出站 `outbound>>>proxy>>>traffic` 统计值。

累计流量会保存到本地 settings 中，关闭应用后再次打开不会清零。

## 配置保存位置

主要配置保存到：

```text
%APPDATA%\NaiwaProxy\settings.json
```

运行时生成的文件：

| 文件 | 说明 |
|------|------|
| `%APPDATA%\NaiwaProxy\config.json` | Xray 运行配置 |
| `%APPDATA%\NaiwaProxy\proxy.pac` | PAC 模式脚本 |
| `%APPDATA%\NaiwaProxy\sing-box-tun.json` | TUN 模式 sing-box 配置 |

## 当前限制

| 项目 | 状态 |
|------|------|
| 协议支持 | 当前主要支持 VMess |
| Real 测速 | 已移除，仅保留 TCP 测速 |
| 订阅自动更新 | 尚未实现 |
| 订阅剩余流量/到期时间 | 尚未实现 |
| 导出节点 | 入口保留，尚未实现 |
| 节点维护 | 入口保留，尚未实现 |
