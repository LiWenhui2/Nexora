# Nexora 产品外观风格文档

本文档用于指导 Stich 重新设计 Nexora 桌面软件 UI。整体方向是现代化、专业、清晰、高效，适合 Windows 桌面网络工具，而不是营销型网页或过度装饰的仪表盘。

## 1. 设计关键词

- 现代桌面工具
- 高信息密度
- 清晰连接状态
- 克制科技感
- 可信赖的网络控制台
- 轻量、干净、稳定
- 面向长期使用

## 2. 视觉定位

Nexora 应像一款成熟的 Windows 网络控制台：界面干净、层级明确、操作直接。用户打开应用后第一眼应看到“当前连接状态”和“当前节点是否可用”，随后能快速进入节点、路由、日志等具体任务。

不要做成夸张的 VPN 营销产品，也不要使用过多发光、渐变、玻璃拟态或大型插画。Nexora 的重点是控制、诊断和效率。

## 3. 整体布局

### 3.1 桌面端主框架

推荐尺寸基准：

- 默认窗口：1460 x 860。
- 最小窗口：1280 x 780。
- 适配 Windows 10/11 桌面环境。

推荐结构：

- 左侧导航栏：宽 248-280px，保持一二级目录结构。
- 顶部状态栏：高度 72-88px。
- 主内容区：表格、面板、表单。
- 右侧抽屉：用于节点编辑、规则编辑、详情预览。

### 3.2 左侧导航层级

左侧导航必须保留当前项目的状态：一级目录为分组标题，二级目录为具体页面入口。Stich 可以重新设计视觉样式，但不要改成扁平列表、顶部 Tab 或单层菜单。

推荐结构：

| 一级目录 | 二级菜单 |
| --- | --- |
| 节点管理 | 节点列表 |
| 节点管理 | 新建节点 |
| 节点管理 | 导入节点 |
| 节点管理 | 导出节点 |
| 节点管理 | 节点测试 |
| 系统 | 设置 |
| 系统 | 日志 |
| 系统 | 关于 |
| 系统 | 检查更新 |
| 系统 | GitHub |

视觉要求：

- 一级目录是小号分组标题，使用次级文字色和较强字重。
- 二级菜单是可点击导航项，左侧保留统一线性图标。
- 当前选中项使用浅琥珀背景、强调色文字和轻微左侧强调线。
- 二级菜单相对一级标题有缩进，保持现有层级感。
- 分组之间留出更大的垂直间距。
- 账号卡片和当前节点摘要仍放在侧栏底部区域。

### 3.3 布局原则

- 总览页可使用卡片，但节点列表、日志、设置页应偏向工具型布局。
- 不要在卡片里再嵌套卡片。
- 主操作固定在可预期位置，例如顶部右侧或面板标题右侧。
- 表格操作与筛选条件应靠近表格。
- 长表单应分组展示，避免单屏堆满。

## 4. 色彩系统

### 4.1 推荐主色

当前项目使用暖黄色作为强调色。新设计可以保留品牌识别，但需要更现代、更克制。

| 主文本 | #111827 | 高可读主文字 |
| 次文本 | #64748B | 描述、标签、弱信息 |

## 5. 字体与字号

推荐字体：

- 中文与界面：Microsoft YaHei UI、Segoe UI。
- 数字、端口、日志：Cascadia Mono、Consolas。

字号建议：

| 场景 | 字号 |
| --- | --- |
| 页面标题 | 24-28px |
| 区块标题 | 18-20px |
| 表格正文 | 13-14px |
| 表单标签 | 13-14px |
| 普通正文 | 14-16px |
| 侧栏导航 | 15-16px |
| 顶部状态数字 | 16-20px |
| 日志文本 | 12-13px |

字重：

- 标题：600-700。
- 导航选中：600。
- 正文：400。
- 关键状态：600。

## 6. 图标风格

使用线性图标，建议风格接近 Lucide/Fluent：

- 节点列表：list。
- 新建：plus。
- 导入：download/upload 或 inbox。
- 导出：share/export。
- 测试：activity/zap。
- 设置：settings。
- 日志：file-text。
- 关于：info。
- 更新：refresh-cw。
- 系统代理：shield。
- TUN：network。
- 路由：route。
- 复制：copy。
- 打开目录：folder-open。
- 删除：trash。

图标统一 16/18/20px，线宽一致。不要混用填充图标和线性图标。

## 7. 组件规范

### 7.1 按钮

按钮分为：

- 主按钮：用于启动代理、导入、保存、检查更新。
- 次按钮：用于测速、复制、打开目录、重置。
- 危险按钮：用于删除节点、移除不可用、清空。
- 图标按钮：用于刷新、更多、复制、打开目录。

样式要求：

- 圆角 8px。
- 高度 32-40px。
- 主按钮使用强调色，不要全界面滥用。
- 危险操作需要二次确认或明显的危险色。

### 7.2 开关

用于二元状态：

- 代理开关。
- TUN 开关。
- 开机自启。
- 开机静默启动。
- 允许局域网访问。
- 仅显示可用节点。

开关必须有文字状态，不只依赖颜色。

### 7.3 状态徽标

常用状态：

- 运行中：绿色。
- 已停止：灰色。
- 启动中/测试中：蓝色或琥珀色。
- 可用：绿色。
- 待测速：灰色。
- 不可用/失败：红色。
- 高延迟：琥珀色。

徽标圆角 999px，高度 22-26px，文字 12-13px。

### 7.4 表格

节点列表需要高可读表格：

- 表头固定。
- 行高 44-52px。
- 支持选中态和 hover 态。
- 活动节点使用小圆点、徽标或左侧强调线标记。
- 延迟字段用颜色区分：低延迟绿色，中等琥珀，高延迟红色，不可用灰色。
- 表格上方放置筛选器和批量操作。

### 7.5 表单

表单要求：

- 标签在输入框上方或左侧，保持统一。
- 必填字段有明确标识。
- 错误信息显示在字段附近。
- 高级字段可折叠，例如 TLS、Host、SNI、Path。
- 节点编辑建议使用右侧抽屉，宽度 460-560px。

### 7.6 弹窗与抽屉

推荐：

- 登录/注册：居中弹窗。
- 删除确认：小弹窗。
- 节点编辑：右侧抽屉。
- 自定义路由：大弹窗或独立路由页。

弹窗不要过度阴影，背景遮罩透明度约 32%-44%。

## 8. 页面设计指南

## 8.1 总览页

总览页应是“连接控制台”。

布局建议：

- 顶部大状态区域：当前代理状态、当前节点、主开关。
- 第二行：流量统计四项，当前下载、当前上传、今日流量、累计流量。
- 下方两列：最近节点状态、最近诊断/日志。
- 右侧或底部：快捷操作。

视觉重点：

- 主开关是页面最强元素。
- 运行中使用绿色状态，但不要把整页染绿。
- 错误状态要显眼，给出“查看日志”和“重试”。

## 8.2 节点页

节点页应是最高信息密度页面。

布局建议：

- 顶部：搜索、地区、协议、订阅、排序、仅可用。
- 表格主体：节点列表。
- 表格右上：新建、导入、全部测速、维护。
- 分组行：订阅名称、节点数量、刷新按钮、定时刷新菜单。
- 右侧抽屉：选中节点详情或编辑表单。

空状态：

- 标题：暂无节点。
- 描述：导入订阅或手动新建节点后即可连接。
- 操作：导入节点、新建节点。

## 8.3 导入页

导入页应降低用户理解成本。

布局建议：

- 输入区占主要宽度。
- 顶部展示支持类型标签：vmess、vless、trojan、ss、socks、http、订阅 URL、二维码。
- 操作按钮：粘贴剪贴板、读取文本文件、扫描二维码图片、识别剪贴板二维码、导入。
- 导入结果在右侧或下方显示。

## 8.4 导出页

导出页应强调“选择节点 -> 获取分享信息”。

布局建议：

- 左侧选择节点和分享链接。
- 右侧二维码预览。
- 操作：复制链接、保存文本、复制二维码、保存二维码。

## 8.5 测试页

测试页应像诊断面板。

布局建议：

- 顶部显示当前节点与代理状态。
- 网站测试使用卡片网格或紧凑列表。
- 每个网站展示图标、名称、URL、状态、延迟、测试按钮。
- “全部测试”和“重置”固定在页面右上。

## 8.6 路由页

路由页应让用户理解不同策略的影响。

布局建议：

- 顶部使用分段控件展示五种模式。
- 每种模式下显示简短说明和影响范围。
- 自定义模式显示六个规则输入区。
- 规则输入区可用三列布局：代理、直连、阻断；每列包含域名和 IP。

## 8.7 设置页

设置页应分组清晰。

推荐分组：

- 启动与后台：开机自启、静默启动、关闭到托盘。
- 网络端口：HTTP、SOCKS、API、允许局域网访问。
- 代理模式：系统代理、PAC。
- TUN：启用状态、运行时检查、权限提示。
- 账号服务：Auth API 地址。
- 数据目录：配置目录、日志目录、运行时目录。

## 8.8 日志页

日志页应像开发者控制台，但保持可读。

布局建议：

- 顶部：级别过滤、类型过滤、搜索、打开目录。
- 主体：等宽日志窗口。
- 日志行按级别轻微着色。
- 错误级别可以在左侧显示红色标记。

## 8.9 登录/注册

登录/注册弹窗应简洁：

- 标题清晰。
- 邮箱和密码输入。
- 错误提示靠近提交按钮。
- 注册页增加验证码和确认密码。
- 已登录状态在侧栏卡片中展示。

## 8.10 关于页

关于页应是信息型面板：

- 应用信息。
- 系统信息。
- 目录信息。
- 许可信息。

目录项右侧使用图标按钮打开文件夹。

## 9. 交互与动效

动效保持轻量：

- 页面切换：120-180ms 淡入或横向轻微位移。
- 开关切换：150ms。
- 表格 hover：背景色过渡 100ms。
- 测试中：使用小型 spinner 或进度点。
- 下载更新：进度条平滑变化。

不要使用大面积弹跳、旋转、粒子、炫光动效。

## 10. 响应式与窗口缩放

虽然是桌面软件，也需要适配窗口尺寸：

- 1280px 宽以下，隐藏部分非关键列或允许横向滚动。
- 左侧导航可缩为图标模式，但展开状态必须保持“一级分组 + 二级菜单”的层级。
- 顶部状态条在空间不足时换行或折叠次要信息。
- 表格列宽应支持内容裁剪和 Tooltip。
- 日志和节点表格必须保留可滚动区域。

## 11. 可访问性

- 文本对比度满足常规阅读。
- 状态不能只依赖颜色，必须有文字。
- 所有图标按钮需要 Tooltip。
- 输入框和按钮需要明显焦点态。
- 错误提示使用红色加文字说明。
- 支持键盘 Tab 顺序。

## 12. 品牌表达

### 12.1 Logo 与名称

- 产品名：Nexora。
- 副标题可用：Windows Xray Client。
- Logo 可使用抽象网络节点、盾牌、路径或“N”字母符号。
- Logo 不要过度复杂，适合托盘、小图标和窗口标题。

### 12.2 语气

界面文案应偏工程化和可信：

- 启动代理
- 停止代理
- 当前节点
- 系统代理
- 路由规则
- 查看日志
- 需要管理员权限
- 未识别到支持的节点链接

避免：

- “极速上网”
- “突破限制”
- “一键起飞”
- 夸张宣传语

## 14. 必须产出的 UI 画面

Stich 至少需要生成：

- 总览页：运行中状态。
- 总览页：启动失败状态。
- 节点页：有多个订阅分组和节点数据。
- 节点页：空状态。
- 节点编辑抽屉。
- 导入页：输入订阅和导入结果。
- 导出页：分享链接与二维码。
- 测试页：网站测试成功/失败混合状态。
- 路由页：自定义规则编辑。
- 设置页。
- 日志页。
- 登录弹窗。
- 关于页。

## 15. 视觉禁忌

- 不要做成官网落地页。
- 不要使用大面积紫蓝渐变或霓虹发光。
- 不要使用低信息密度的大卡片堆叠。
- 不要让主按钮、状态色、警告色混用。
- 不要让表格字段过度居中导致难以扫描。
- 不要隐藏关键网络状态。
- 不要只用图标表达危险操作。
- 不要在日志页使用花哨背景。
