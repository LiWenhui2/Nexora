# Nexora 桌面客户端功能说明文档

本文档用于交付给 Stich 重新设计 Nexora 桌面软件 UI。目标是基于当前项目能力，梳理页面结构、核心功能、交互流程与状态展示要求，帮助设计工具生成完整、可落地的现代化桌面端界面。

## 1. 产品定位

Nexora 是一款面向 Windows 用户的桌面代理客户端。它负责管理代理节点、导入订阅、启动和停止代理核心、配置系统代理、开启 TUN 模式、查看流量统计、测试节点和网站连通性，并提供日志诊断、更新检查与账号登录能力。

产品应强调三件事：

- 连接状态清晰：用户一眼知道当前是否已连接、正在使用哪个节点、系统代理/TUN/路由模式是否生效。
- 节点管理高效：用户能快速导入、筛选、测速、切换、维护节点和订阅分组。
- 网络诊断可信：当代理不可用、节点延迟高、订阅失败、TUN 权限不足时，界面能给出明确状态和下一步动作。

## 2. 目标用户

- 需要长期使用代理网络的 Windows 桌面用户。
- 需要管理多协议、多订阅、多地区节点的高级用户。
- 需要通过日志、测速、路由规则排查网络问题的技术用户。
- 需要一键连接、不想理解复杂配置的普通用户。

## 3. 核心能力概览

| 模块 | 功能范围 |
| --- | --- |
| 连接控制 | 启动/停止代理、选择当前节点、显示运行状态、验证代理可用性 |
| 节点管理 | 新建、编辑、导入、导出、删除、筛选、排序、测速、去重、移除不可用节点 |
| 订阅管理 | URL/内容导入、订阅分组、刷新订阅、定时刷新、显示节点数量与流量信息 |
| 系统代理 | 自动配置 Windows HTTP 代理、PAC 模式、清除代理、不修改系统代理 |
| TUN 模式 | 使用 sing-box 与 Wintun 转发系统流量，提示管理员权限 |
| 路由规则 | 全局代理、绕过大陆、绕过局域网、直连、自定义代理/直连/阻断规则 |
| 流量统计 | 当前上传/下载速度、今日流量、累计流量、订阅剩余流量 |
| 连通性测试 | TCP 节点延迟测试、常用网站通过代理访问测试 |
| 日志诊断 | 系统日志、流量日志、INFO/WARN/ERROR/CRASH 过滤、打开日志目录 |
| 账号 | 邮箱登录、注册、验证码、退出登录、保存会话 |
| 更新与关于 | 检查 GitHub Release、下载进度、应用/系统/Core/TUN/目录/许可信息 |
| 托盘 | 后台运行、托盘启动/停止、切换节点、切换路由、打开主窗口、退出 |

## 4. 信息架构

建议新 UI 采用左侧分组导航 + 顶部连接控制条 + 主内容区的桌面应用结构。左侧导航必须保持当前项目的一二级目录形态：一级目录作为分组标题，二级目录作为可点击页面入口，不要改成扁平导航，也不要把所有页面合并成顶部 Tab。

### 4.1 左侧导航层级

当前导航结构应保留为：

| 一级目录 | 二级入口 | 说明 |
| --- | --- | --- |
| 节点管理 | 节点列表 | 管理节点、订阅分组、筛选、排序、测速、维护 |
| 节点管理 | 新建节点 | 手动创建节点，可用独立页面、内嵌表单或右侧抽屉承载 |
| 节点管理 | 导入节点 | 从链接、订阅、文件、剪贴板、二维码导入 |
| 节点管理 | 导出节点 | 导出分享链接和二维码 |
| 节点管理 | 节点测试 | 网站连通性测试和当前节点测试 |
| 系统 | 设置 | 启动项、系统代理、TUN、局域网访问、端口等设置 |
| 系统 | 日志 | 查看系统日志和流量监控日志 |
| 系统 | 关于 | 应用、系统、目录、许可信息 |
| 系统 | 检查更新 | 检查和下载 GitHub Release 更新 |
| 系统 | GitHub | 打开项目主页 |

### 4.2 导航保留要求

- 保持“节点管理”“系统”两个一级分组标题。
- 保持“节点列表、新建节点、导入节点、导出节点、节点测试、设置、日志、关于、检查更新、GitHub”这些二级入口。
- 一级分组不可点击，仅用于视觉分组和信息层级。
- 二级入口应有清晰选中态、hover 态和图标。
- 可以优化样式、间距、图标和选中态，但不要改变导航层级关系。
- 页面内部可以使用 Tab、抽屉或弹窗提升效率，但不能替代左侧二级导航入口。

## 5. 全局布局要求

### 5.1 顶部连接控制条

顶部应固定显示关键状态和高频操作：

- 当前代理总开关：未运行、启动中、运行中、停止中、启动失败。
- 当前节点选择器：显示节点名、协议、地区、地址端口、延迟。
- 当前流量速率：下载速度、上传速度。
- 系统代理模式：自动配置、PAC、清除、不修改。
- 路由模式：全局、绕过大陆、绕过局域网、直连、自定义。
- TUN 开关：关闭、启动中、运行中、需要管理员权限、运行异常。

### 5.2 左侧导航区

左侧导航应显示：

- 产品 Logo 与名称：Nexora。
- 一级分组标题：节点管理、系统。
- 二级菜单入口：节点列表、新建节点、导入节点、导出节点、节点测试、设置、日志、关于、检查更新、GitHub。
- 账号卡片：未登录时显示“登录/注册”；已登录时显示邮箱与退出登录。
- 当前节点摘要：节点名、延迟、运行状态。

### 5.3 主内容区

主内容区应按任务聚焦展示，不建议堆叠太多大卡片。桌面工具应更偏向高信息密度、清晰表格和可扫描状态。

## 6. 页面功能说明

## 6.1 总览页

总览页用于展示当前代理状态和常用操作。

### 核心内容

- 连接状态卡：显示代理是否运行、运行时长、当前节点、系统代理状态、TUN 状态。
- 当前节点卡：节点名称、协议、地区、地址端口、TCP 延迟、可用性。
- 流量统计：当前下载、当前上传、今日下载、今日上传、累计下载、累计上传。
- 快捷操作：启动/停止代理、当前测速、全部测速、最快节点、刷新订阅、打开日志。
- 最近异常：展示最近一次启动失败、验证失败、订阅失败或 TUN 错误。

### 状态要求

- 未选择节点时，启动按钮禁用，并提示先导入或选择节点。
- 启动中显示进度感，不应让用户重复点击。
- 启动失败时突出错误摘要，并提供“查看日志”按钮。

## 6.2 节点页

节点页是核心工作区，用于管理所有代理节点。

### 节点列表字段

- 名称
- 协议
- 地区
- 地址
- 端口
- 延迟
- 可用性
- 订阅来源
- 到期时间
- 总流量
- 剩余流量
- 更新时间

### 节点操作

- 设为活动节点。
- 新建节点。
- 编辑节点。
- 删除节点。
- TCP 测速。
- 全部测速。
- 移除不可用节点。
- 去除重复节点。
- 订阅分组刷新。
- 设置订阅定时刷新：关闭、5 分钟、15 分钟、30 分钟、1 小时、6 小时、12 小时、24 小时、自定义。

### 筛选与排序

- 按地区筛选。
- 按协议筛选：VMess、VLESS、Trojan、Shadowsocks、SOCKS、HTTP。
- 按订阅来源筛选。
- 仅显示可用节点。
- 默认排序与延迟优先排序。

### 分组要求

- 手动节点归为“手动”分组。
- 订阅导入的节点按订阅来源分组。
- 每个分组显示节点数量。
- 非手动分组支持刷新和定时刷新。

## 6.3 新建/编辑节点

节点编辑可使用弹窗、右侧抽屉或内嵌表单。建议使用右侧抽屉，避免离开节点列表。

### 字段

- 协议：vmess、vless、trojan、shadowsocks、socks、http。
- 名称。
- 地址。
- 端口。
- UUID/用户名。
- 密码。
- Alter ID。
- Security：auto、none、aes-128-gcm、aes-256-gcm、chacha20-poly1305、2022-blake3-aes-128-gcm、2022-blake3-aes-256-gcm。
- Network：tcp、ws、grpc。
- TLS：none、tls。
- Host。
- SNI。
- Path。

### 校验

- 地址不能为空。
- 端口必须为 1-65535。
- VMess/VLESS 需要 UUID。
- Trojan/Shadowsocks 需要密码。
- SOCKS/HTTP 可选用户名和密码。

## 6.4 导入页

导入页用于从多种来源导入节点和订阅。

### 支持输入

- 单条或多条分享链接。
- 订阅 URL。
- Base64 订阅内容。
- 本地文本文件。
- 剪贴板文本。
- 二维码图片。
- 剪贴板二维码图片。

### 支持协议

- vmess://
- vless://
- trojan://
- ss://
- socks:// 或 socks5://
- http:// 或 https:// 代理节点

### 导入结果

导入完成后应显示：

- 成功导入数量。
- 跳过/失败数量。
- 订阅来源名称。
- 是否读取到订阅流量信息。
- 是否存在重复节点。
- 后续动作：查看节点、全部测速、继续导入。

## 6.5 导出页

导出页用于导出单个节点分享信息。

### 功能

- 选择要导出的节点。
- 显示分享链接。
- 复制分享链接。
- 保存为文本文件。
- 显示二维码。
- 复制或保存二维码图片。

### 状态

- 未选择节点时，分享链接和二维码区域显示空状态。
- 不支持导出的协议应给出明确提示。

## 6.6 测试页

测试页用于验证当前节点和代理链路是否可用。

### 当前节点测试

- 显示当前节点名称。
- 显示代理运行状态。
- 支持单站点测试和全部站点测试。
- 支持重置测试结果。

### 默认网站

- GitHub
- ChatGPT
- YouTube
- Google
- Twitter
- Wikipedia
- Discord
- Telegram

### 测试结果字段

- 网站名称。
- URL。
- 图标。
- 状态：待测试、测试中、成功、失败、超时。
- 延迟。
- HTTP 状态或错误摘要。

## 6.7 路由页

路由页用于配置代理策略。

### 内置模式

- 全局代理：默认通过当前节点代理所有流量。
- 绕过大陆：大陆 IP 和域名直连，其他走代理。
- 绕过局域网：局域网和私有地址直连，其他走代理。
- 直连模式：所有流量直连。
- 自定义规则：按用户规则代理、直连或阻断。

### 自定义规则

自定义规则分为六个输入区：

- 代理域名。
- 直连域名。
- 阻断域名。
- 代理 IP/CIDR。
- 直连 IP/CIDR。
- 阻断 IP/CIDR。

### 规则输入格式

域名支持：

```text
example.com
domain:example.com
full:example.com
regexp:^.*\.example\.com$
geosite:category-ads-all
```

IP 支持：

```text
1.2.3.4
1.2.3.0/24
geoip:cn
```

### 交互要求

- 自定义模式下才显示“编辑规则”入口。
- 保存自定义规则后应自动启用自定义路由。
- 如果代理正在运行，切换路由模式后应提示配置已更新，必要时自动重启或提示用户重启代理。

## 8. 关键用户流程

### 8.1 首次使用流程

1. 打开应用。
2. 空状态提示“导入节点或订阅”。
3. 用户进入导入页，粘贴订阅 URL 或节点链接。
4. 导入成功后进入节点页。
5. 用户点击全部测速。
6. 用户选择一个可用节点。
7. 用户打开代理开关。
8. 应用启动 Core、配置系统代理、验证代理可用性。
9. 顶部状态变为运行中。

### 8.2 日常连接流程

1. 用户打开应用。
2. 应用加载上次选择的节点与系统代理设置。
3. 用户直接打开代理开关。
4. 如设置了启动自动连接，则自动启动代理。
5. 用户通过顶部状态查看当前流量和延迟。

### 8.3 订阅刷新流程

1. 用户在节点页找到订阅分组。
2. 点击刷新。
3. 应用拉取订阅 URL。
4. 解析节点与订阅流量信息。
5. 更新该分组节点。
6. 显示刷新结果。
7. 可选择立即测速。

### 8.4 故障排查流程

1. 用户发现代理无法访问。
2. 总览页显示验证失败或当前节点不可用。
3. 用户进入测试页运行常用网站测试。
4. 用户进入日志页查看 ERROR/CRASH。
5. 用户切换节点、切换路由或重新启动代理。

## 9. 状态与异常场景

| 场景 | UI 要求 |
| --- | --- |
| 无节点 | 展示空状态，引导导入节点 |
| 未选择节点 | 启动按钮禁用或点击后提示选择节点 |
| Core 文件缺失 | 显示运行时缺失，提供打开运行时目录 |
| 节点启动失败 | 展示错误摘要和查看日志入口 |
| 代理验证失败 | 显示验证失败，但保留停止/重试操作 |
| 系统代理被外部修改 | 提示系统代理状态不一致，可重新应用 |
| TUN 权限不足 | 提示需要管理员权限 |
| TUN 运行时缺失 | 提示缺少 sing-box.exe 或 wintun.dll |
| 订阅 URL 失败 | 显示 HTTP 状态或网络错误 |
| 二维码识别失败 | 提示未识别到支持的节点链接 |
| 日志为空 | 显示空状态和日志目录入口 |
| 更新失败 | 提示检查网络或切换全局代理后重试 |

## 12. 文案基调

- 简洁、清楚、工程感，不使用夸张营销文案。
- 状态文案应直接：运行中、已停止、启动失败、待测速、可用、不可用、超时。
- 错误提示应包含下一步：查看日志、重试、切换节点、检查运行时、以管理员身份运行。
